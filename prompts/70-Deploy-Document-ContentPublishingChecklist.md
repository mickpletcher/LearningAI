# Prompt 70

You are working inside an Astro website project called Learn AI the American Way.

Your task:
Create a practical content publishing checklist for adding new pages, lessons, and content updates to the site.

Goal:
Help a future maintainer publish new content in a consistent, low error way without needing deep Astro expertise.

Files to review and update:

- README.md

- MAINTENANCE.md or docs/maintenance-workflow.md if it exists

- LAUNCH-CHECKLIST.md or docs/launch-checklist.md if it exists

- optionally create a new file if appropriate:

  - docs/content-publishing-checklist.md

  - or CONTENT-PUBLISHING-CHECKLIST.md in the project root

Checklist topics to include:

1. writing review

2. title and intro review

3. internal link review

4. navigation update review if needed

5. metadata review

6. mobile readability review

7. trust and safety alignment review

8. lesson index or use case hub update review if relevant

9. local preview review

10. deployment verification after publish

Suggested checklist items:

- confirm the page has a clear purpose

- confirm the title and intro are strong

- confirm links work

- confirm page tone matches the site

- confirm prompts are practical and safe

- confirm no placeholder text remains

- confirm lessons are indexed if needed

- confirm navigation only changes when necessary

- confirm the site builds locally

- confirm the live page works after deployment

Documentation requirements:

- write in plain English

- keep it step by step

- make it practical and short enough to actually use

- align it with the existing maintenance and launch docs

- explain where to update content collections or index pages if relevant

Tone requirements:

- practical

- direct

- clear

- calm

- not robotic

- not corporate

Important constraints:

- do not duplicate the entire maintenance or launch guide

- do not create bloated editorial process docs

- do not assume the maintainer knows Astro internals

- do not make the checklist so long that it becomes useless

Implementation requirements:

- keep the checklist aligned with the actual project structure

- link it from existing maintenance docs if helpful

- make it easy to update later

- keep the process light but reliable

Output requirement:
Generate the actual updated contents for the relevant documentation files and any new content publishing checklist file if needed.
Do not summarize.
Do not explain.
Create the content publishing checklist.
