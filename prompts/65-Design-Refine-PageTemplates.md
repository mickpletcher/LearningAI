# Prompt 65

You are working inside an Astro website project called Learn AI the American Way.

Your task:
Refine the site’s page templates and shared layout patterns so pages feel structurally consistent while still allowing room for content differences.

Goal:
Make the site feel more polished and easier to maintain by improving common page structure patterns.

Files to review and update:

- src/layouts/BaseLayout.astro

- shared page level components in src/components/

- any repeated page structure patterns in src/pages/

- src/styles/global.css or the current global stylesheet location

Template and structure goals:

1. improve consistency of page headers

2. improve consistency of intro sections

3. improve consistency of section spacing

4. improve consistency of page endings and next step areas

5. improve how prompt boxes and watch out sections fit into the layout

6. make pages easier to maintain as the site grows

Tasks:

- identify repeated layout or structure patterns that should be standardized

- improve shared wrappers, spacing, and headings if needed

- create or refine reusable patterns where they truly help

- make sure the layout supports beginner readability

- improve visual rhythm from page to page

- keep enough flexibility so pages do not all feel identical

Design and UX guidance:

- clean and simple

- readable

- practical

- consistent

- calm

- not flashy

- not overdesigned

- easy to maintain

Important constraints:

- do not overabstract the project into too many components

- do not create unnecessary complexity

- do not make every page look mechanically identical

- do not break existing content or working links

- do not redesign the whole site just for the sake of uniformity

Implementation requirements:

- directly update the relevant layout, shared components, and styles

- preserve Astro structure and GitHub Pages compatibility

- keep the system lightweight and maintainable

- improve structure only where it clearly helps

Output requirement:
Generate the actual updated file contents for the relevant layout, shared component, page, and style files where needed.
Do not summarize.
Do not explain.
Refine the site’s page templates.
