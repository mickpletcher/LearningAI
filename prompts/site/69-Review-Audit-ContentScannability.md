# Prompt 69

You are working inside an Astro website project called Learn AI the American Way.

Your task:
Audit the site's content for scannability and improve how quickly beginners can understand each page.

Goal:
Make the site easier to skim, scan, and use without weakening the educational value.

Files to review:

- all files in src/pages/

- all Markdown files in src/content/lessons/

- shared components that shape content presentation such as page intros, prompt boxes, and card sections

Audit focus:

1. heading usefulness

2. section length

3. paragraph density

4. list quality

5. whether important ideas are buried

6. whether prompt examples stand out clearly

7. whether next steps are easy to find

8. whether pages feel tiring to scan on desktop and mobile

Tasks:

- improve headings that are vague or generic

- break up dense sections

- tighten long paragraphs

- improve use of short lists where helpful

- move buried practical advice higher when needed

- make important examples and prompts easier to spot

- improve page endings so the next action stands out

- improve rhythm and pacing across long pages

Tone requirements:

- plainspoken

- practical

- calm

- direct

- beginner friendly

- not robotic

- not corporate

- no hype

Important constraints:

- do not replace all paragraphs with bullet lists

- do not oversimplify useful content into shallow snippets

- do not remove helpful context just to shorten a page

- do not produce an audit report without making the fixes directly

- do not rewrite everything unnecessarily

Implementation requirements:

- directly update the relevant files

- preserve Astro structure and working links

- keep the site readable, useful, and visually balanced

- ensure scannability improvements help both desktop and mobile users

Output requirement:
Directly update the relevant files to improve content scannability.
Do not summarize.
Do not explain.
Fix the site’s scannability.
